EVOLUCION DEL DESARROLLO WEB

El desarrollo web ha recorrido un largo camino desde sus comienzos: Década de 1990: La web comenzó con páginas estáticas en HTML, compuestas principalmente por texto y enlaces simples.

Década de 2000: La incorporación de CSS permitió separar el contenido de la presentación, y JavaScript introdujo la interactividad.
Aparecieron los primeros sistemas de gestión de contenidos (CMS) como WordPress.

Década de 2010: La web se volvió más dinámica gracias a frameworks y bibliotecas como AngularJS, React y Vue.js. El diseño responsivo se volvió esencial para adaptar las páginas a diferentes dispositivos. Década de 2020 en adelante: Se ha visto un gran impulso en el desarrollo backend con entornos como Node.js.
La inteligencia artificial y la automatización comenzaron a integrarse en la experiencia web.

En resumen, hemos pasado de simples páginas estáticas a aplicaciones web complejas y altamente interactivas.

ARQUITECTURA DE SERVICIOS

Cliente-Servidor es uno de los estilos arquitectónicos distribuidos más conocidos, el cual está compuesto
por dos componentes, el proveedor y el consumidor. El proveedor es un servidor que brinda una serie de servicios
o recursos los cuales son consumido por el Cliente.

En una arquitectura Cliente-Servidor existe un servidor y múltiples clientes que se conectan al servidor para recuperar
todos los recursos necesarios para funcionar, en este sentido, el cliente solo es una capa para representar los datos y
se detonan acciones para modificar el estado del servidor, mientras que el servidor es el que hace todo el trabajo pesado.
